Tutorial

A brief walkthrough of usage.

The tasks that follow should give you an introduction to how to progress through levels. Each of these should take anywhere from a few seconds to a few minutes. If you're having difficulty with something, feel free to submit an issue on Github. If you're repeatedly having with these tasks, it might be worthwile to pause for a bit and refresh your basic Python skills before moving on with Excision. We'll walk through our hello_world.py example together, then they'll be some practice problems for you to try.

1. Hello World!

Imagine you're given the following task:

Problem:
Create a program that writes "Hello World!" to Excision/results/tutorial/hello.txt (without the quotes, but with a newline character).

Take a second to make sure you could do that. You need to know where you installed Excision and how to write to a file. Virtually all of the programs you write will dump their output to the Excision/results directory. Here's how I accomplished the task. Notice that I'm using UNIX paths. I'll continue that convention throughout Excision. If you're on Windows, make sure to use Windows paths. The bioinformatics equivalent of "Hello World!" is finding GC content...

2. Submitting Scripts

Try completing this problem:
Problem:
Create a program that takes a nucleotide sequence as an arguments and writes the GC content to Excision/results/tutorial/gc.txt to the nearest whole percent.

Example:
$ ./gc_calc.py ATATGCGC

Result:
50%

Once you've written this script, you can test it out with the 'Submit' button. Just to see what happens, you should make sure to submit an incorrect script first.

3. Hints

Besides a submit button, the other thing you'll see at each question are hint(s). Hint's will give you little prompts on how to go about solving the problem at hand. Hints aren't free though. Each hint will require you to solve a secondary problem. These problems will be small, generally just a few of lines of code. So, it'll be up to you whether you think taking the time to solve the hint is worthwhile. In general, the hints are less likely to be strictly bioinformatics related. Instead, they'll focus on introducing you to new parts of Python and its ecosystem that might come in handy farther down the road.

Let's try another problem. This one is a bit harder, so it might be worthwhile to solve the hints. Even if you know how to solve this task, take a second to make sure you know what to expect from hints.

Task: Write a function called squared_sum. It should take a list as a parameter and returns a float. squared_sum should find the sum of squares of the list.

{% if hint_solved1 %}

This is a multi-part hint:

  1. Notice the form of this task, as opposed to the main questions. Code for hints will generally require you to write a function instead of a script.
  2. Besides the concept of hints, the second purpose of this question is to introduce you to the concept of using modules. You can easily without importing anything. But the line import numpy as np will make the question significantly easier. You'll want to import some module in virtually every follwoing question.

{% endif %}

Task: Write a function called seen. It should take two lists as parameters and return a third list. The returned list should contain only the unique items in the second list. That is, the elements that are in the second list but not the first list. If there are redundant items in the second list (i.e. [1, 1]), only include that item in the third list once.

{% if hint_solved2 %}

We'll introduce other ways to read data files, but numpy provides some basic help. Read up on np.loadtxt to get your data as an array. Once you have the array, you have access to the power of numpy. There are a lot of things which can be done with arrays. Some of the most common features are well introduced in the quickstart of the offical numpy documentation. The function that will help you most at the moment is sum.

{% endif %}
Problem:
Create a program that reads a csv file of numbers. Find the largest sum of any row or column. Write the result, to the nearest integer, in Excision/results/tutorial/sum.txt.

Notes:
The correct answer is guaranteed to be unique. So you don't have to worry about dealing with a tie.

Example:
Contents of /path/to/numbers.csv:
5,1,8,7
4,5,7,8
8,7,1,6
3,7,6,8


$ ./row_col_max.py /path/to/numbers.csv

Result:
29
