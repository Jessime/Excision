Introduction

What follows will give you an introduction to what you can expect as you progress through the game. Each part of the tutorial should take you just a few minutes, though if you have to look up how to do something it might take longer. Googling is fine, even in the tutorial, so do as much research as you need. If you're having difficulty with something, feel free to submit an issue on Github. If you're repeatedly having trouble with these problems, it might be worthwile to pause for a bit and refresh your fundamental Python skills before moving on with Excision. We will walk through an example problem together; then, there will be some practice problems for you to try.

Hello, World!

Imagine you're given the following task:

Create a program that writes "Hello, World!" to Excision/results/tutorial/hello.txt (without the quotes, but with a newline character).

Take a second to make sure you could do that. You need to know...

  1. where you installed Excision,
  2. how to write a string to a file, and
  3. what a newline character is.

Hints

Besides a submit button, the other thing you'll see at each question are hints. Hints will give you little prompts on how to go about solving the problem at hand. Hints aren't free though. Each hint will require you to solve a secondary task. These tasks will be small, generally just a few of lines of code. So, it'll be up to you whether you think taking the time to solve for the hint is worthwhile. In general, the hints are less likely to be strictly bioinformatics related. Instead, they'll focus on introducing you to new parts of Python and its ecosystem that might come in handy farther down the road.

Another difference with hints is that you don't have to separate them into their own documents. If you want to have all the code for all of your hints in a single file, that's fine. Or you can put all of the hints for a single level in a file. However you want to do it is fine. Just make sure that, when you submit a file, a function with the proper name exists. It doesn't matter what else is in the file.

Problem

You have three files, all of which are in FASTA format. 1. data/1/reference.fa contains a single sequence. This sequence is what is expected if no mutations are present in a sample. 2. data/1/control.fa contains ~50 sequences of the same region of the genome as the reference sequence. 3. data/1/theraptrix.fa is the file recovered yesterday. It contains another ~50 sequence from people who may have been exposed to illegally dumped waste.

Write a program that, using these three files, calculates the p-value of the difference in mutation rates between control.fa and theraptrix.fa. Output the p-value, to 4 decimal places, to results/1.txt.

Notes:

To make things easy, you can make the following assumptions:

  1. There are only single nucleotide polymorphisms (SNPs) mutations present in the dataset. And all sequences will be the same length. You can compare mutations on a nucleotide by nucleotide basis, without length normalization.

  2. The error rates in both files will satisfy all of the necessary assumptions for a Student's t-test.

Example:

Contents of reference.fa:

                
>reference_sequence
CAGGTCACTT

Contents of control.fa:

                
>c1
CAGGTCCCTT
>c2
CACGCCACTT
>c3
TAGGTCACTT

Contents of theraptrix.fa:

                
>OR1
CAAATCACTA
>OR2
CACGCCACTT
>OR3
CGGGTCACGT

Execution:

$ ./mutations.py /path/to/reference.fa /path/to/control.fa /path/to/theraprix.fa

Result:

0.1011
