Introduction

This tutorial will give you an introduction to what you can expect as you progress through the game. Each part of the tutorial should take you just a few minutes, though if you have to look up how to do something it might take longer. Googling is fine, even in the tutorial, so do as much research as you need. If you're having difficulty with something, feel free to submit an issue on Github. If you're repeatedly having trouble with these problems, it might be worthwile to pause for a bit and refresh your fundamental Python skills before moving on with Excision. We will walk through an example problem together; then, there will be some practice problems for you to try.

Hello, World!

Imagine you're given the following task:

Create a program that writes "Hello, World!" to Excision/results/tutorial/hello.txt (without the quotes).

Take a second to make sure you could do that. You need to know...

  1. where you installed Excision,
  2. how to write a string to a file, and
  3. how to close a file properly.
Virtually all of the programs you write will dump their output to the Excision/results directory. Here's how I solved the problem. Notice that I'm using UNIX paths. I'll continue that convention throughout Excision. If you're on Windows, make sure to use Windows paths.

Tasks and Hints

There will be optional tasks available below the main problem for each level. Your reward for solving each task is a hint to assist you in your completion of the main problem. The tasks will be small (generally just a few of lines of code), so it'll be up to you to decide whether taking the time to solve for the hint is worthwhile. In general, the tasks are less bioinformatics-related than the main problems. Instead, they'll focus on introducing you to new parts of Python and its ecosystem that might come in handy further down the road.

Another difference between the tasks and the problems is that tasks only require you to write a function instead of an executable file. A successful task submission includes a correct function named according to the task instructions. Everything else in the submission is ignored; thus, you can include as many functions in a single document as you'd like, resubmitting the document for multiple tasks. Alternatively, you can create a new document for each function. The choice is yours.

Problem

Create a program that reads a .csv file containing numbers. Find the largest sum of any row or column. Write the result, to the nearest integer, in Excision/results/tutorial/sum.txt.

Note(s):

  • The correct answer is guaranteed to be unique, so you don't have to worry about dealing with a tie.
Example

Contents of /path/to/numbers.csv:

                  
5,1,8,7
4,5,7,8
8,7,1,6
3,7,6,8

Execution:

$ ./row_col_max.py /path/to/numbers.csv

Result:

                  
29

Task

Write a function called squared_sum. It should take a list as a parameter. squared_sum should find the sum of squares of the list and return that sum as a float.

Hint:

This is a multi-part hint:

  1. Notice the form of this task, as opposed to the main questions. Code for hints will generally require you to write a function instead of a script.
  2. Besides the concept of hints, the second purpose of this question is to introduce you to the concept of using modules. You can solve this question without importing anything. But the line import numpy as np will make the question significantly easier. You'll want to import some module in virtually every following question.

Task

Write a function called seen. It should take two lists as parameters and return a third list. The returned list should contain only the unique items in the second list—that is, the elements that are in the second list but not the first list. If there are redundant items in the second list (e.g., [1, 1]), only include a single instance in the third list.

Hint:

We'll introduce other ways to read data files, but numpy provides some basic help. Read up on np.loadtxt to get your data as an array. Once you have the array, you have access to the power of numpy. There are a lot of things which can be done with arrays. Some of the most common features are well introduced in the quickstart of the offical numpy documentation. The function that will help you most at the moment is sum.
